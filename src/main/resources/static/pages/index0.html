《Java EE应用开发》课程学习指导

  • Java EE是指Java Enterprise Edition（Java企业版），早期称为J2EE。
  • Java EE不是编程语言，也并非一个产品，而是一系列的技术规范，并成为企业级开发的首选平台之一。
  • Java EE应用服务器是为应用程序提供运行环境、为组件提供服务。
  • 在小型系统（一台Web服务器）的情况下，通常选择Tomcat作为Java EE应用服务器，而大型系统（多台Web服务器）的情况下，通常选择JBoss作为作为 Java EE应用服务器来运行EJB项目、形成分布式应用。
  • 在Tomcat里运行的一般Java Web项目，也是可以部署到JBoss里运行的。
  • 在Java EE应用系统里，一般需要数据库服务器（如MySQL、SQL Server、Oracle和DB2等）来辅助。
  • 小型项目开发，可以使用JSP技术，参见会员管理项目MemMana1。
  • JavaBean是一种Java语言写成的可重用组件，参见会员管理项目memmana2。
  • 使用JSP技术开发的站点中，每个JSP页面在运行时最终转译为一个Servlet。JSP技术是在Servlet技术之后产生的技术。
  • Servlet是Java EE应用中重要的基础技术，MVC模式开发中担任控制器的角色。Servlet可以完成JSP的所有功能，同时具有JSP不具备的某些功能。使用MVC模式开发的项目，参见会员管理项目memmana3。
  • 使用Servlet或JSP，其开发效率较低，但运行效率高。为了提高项目的开发效率，一般使用MVC框架。如SSM和SSH等。
  • MyBatis是一种ORM产品，可认为是对JDBC的再封装。
  • Spring MVC框架是对Servlet的再封装，使用该框架后，Tomcat将控制权转交给Spring MVC的核心控制器，用来处理用户对控制器方法的请求。同时，减少了控制器文件的数量。
  • Spring作为IoC容器，用于对象创建和依赖关系维护，在分层架构的Web项目里实现解耦。
  • 轻量级的Java EE应用，目前广泛使用SSM整合。其中，Spring是框架整合的核心。
  • Sun公司官方倡导的经典级Java EE应用，以EJB 3为核心来构建业务逻辑层，需要EJB容器支持，运行于JBoss等Web服务器，并且业务层可供客户端或Web层调用，实现系统分布式的应用架构。经典框架则以JSF+EJB+JPA为代表。
